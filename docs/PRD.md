# PRD — 微信公众号文章采集与理论收敛分析工具

## 1. 项目信息

| 项 | 内容 |
|---|---|
| **项目名称** | wechat_article_analyzer |
| **语言** | 中文 |

| **界面形式** | CLI（命令行界面） |

### 原始需求复述

用户章锋（老铁）在微信公众号"复合体理学"上发表了近2000篇文章，涵盖四大理论基石（刘原理、三视界法、太乙预言机、全息拓扑动力学）。需构建工具：通过浏览器自动化登录微信公众号后台批量抓取全部文章，存入SQLite数据库，再利用DeepSeek API进行深度AI分析，实现理论收敛——提取核心概念、构建概念关系图谱、追踪理论演化脉络、生成理论收敛报告。

---

## 2. 产品目标

1. **全量采集**：通过Playwright自动化登录微信公众号后台，批量抓取全部已发布文章（标题、发布时间、正文HTML、正文纯文本、封面图URL），确保零遗漏地存入SQLite数据库。

2. **深度理论分析**：利用DeepSeek API对文章进行逐篇及全局分析，提取核心概念与关键词，识别反复出现的理论支柱和核心命题，构建概念关系图谱。

3. **理论演化追踪**：按时间线追踪理论概念从早期到最近的发展脉络，可视化呈现理论体系的演化路径。

4. **理论收敛报告**：自动生成结构化的理论收敛报告，输出核心理论框架、关键概念集群、概念演化路径，帮助用户收敛2000篇文章的理论精华。

### CLI界面设计

#### 命令总览

```
wechat-article-analyzer <command> [options]
```

#### 命令详情

**1. login — 登录微信公众号后台**

```
python -m src.main login [--timeout 300]
```
- 启动Playwright浏览器，打开mp.weixin.qq.com
- 用户手动扫码登录
- `--timeout`：登录超时时间（秒），默认300秒
- 登录成功后保存登录态（cookies），后续操作可复用

**2. crawl — 批量采集文章**

```
python -m src.main crawl [--resume] [--limit N]
```
- 从素材管理页面批量获取文章列表并逐篇抓取正文
- `--resume`：断点续抓模式，跳过已采集的文章
- `--limit N`：限制采集数量（用于测试），默认不限制
- 实时显示进度：`[150/2000] 正在抓取: 刘原理与三视界法的深层关联...`

**3. analyze — AI分析文章**

```
python -m src.main analyze [--resume] [--article-id N]
```
- 调用DeepSeek API对文章进行逐篇分析
- `--resume`：跳过已分析的文章
- `--article-id N`：只分析指定ID的单篇文章
- 实时显示进度：`[800/2000] 正在分析: 太乙预言机的数学基础...`

**4. report — 生成理论收敛报告**

```
python -m src.main report [--output ./output/report.md]
```
- 基于全量分析结果生成理论收敛报告
- `--output`：报告输出路径，默认 `./output/report.md`
- 报告包含：核心理论框架、关键概念集群、概念演化路径、理论支柱总结

**5. graph — 构建概念关系图谱（P1）**

```
python -m src.main graph [--output ./output/concept_graph.json]
```
- 统计概念共现频率，生成概念关系图谱数据
- `--output`：图谱数据输出路径

**6. status — 查看采集与分析状态**

```
python -m src.main status
```
- 显示数据库统计信息：总文章数、已采集数、已分析数、失败数
- 示例输出：
  ```
  采集状态: 2000/2000 篇已完成, 0 篇失败
  分析状态: 1200/2000 篇已完成, 5 篇失败
  ```

**7. config — 配置管理**

```
python -m src.main config set deepseek_api_key <KEY>
python -m src.main config show
```
- 管理DeepSeek API密钥等配置项
- 配置存储在 `config.json` 或环境变量中

#### 全局选项

## 5. 待确认问题

1. **文章数量确认**：用户提到"近2000篇"，需确认实际文章总数，以评估采集时长和API调用成本。

2. **DeepSeek API额度**：2000篇文章逐篇分析 + 全局报告生成，预计需要约2000+次API调用。需确认用户的DeepSeek API额度是否充足，以及是否有调用频率限制。

3. **概念关系图谱输出格式**：P2-3中提到可视化，用户是否需要交互式可视化（如D3.js网页），还是静态图谱（Mermaid/Graphviz）即可？考虑到CLI定位，默认提供Mermaid/Graphviz格式。

4. **理论支柱预定义**：四大理论基石（刘原理、三视界法、太乙预言机、全息拓扑动力学）是否需要作为预定义标签传入分析Prompt，还是让AI自行识别？建议预定义以提升准确度，需用户确认。

5. **采集频率限制**：微信公众号后台对频繁访问可能有限制。需确认是否需要设置请求间隔（如每篇间隔2-5秒），以及遇到限制时的退避策略。

6. **TOMAS-AGI / 太极OS项目关联**：用户同时推进这两个项目，理论收敛报告是否需要特别标注与这两个项目相关的概念和文章？如需要，需提供相关关键词。
